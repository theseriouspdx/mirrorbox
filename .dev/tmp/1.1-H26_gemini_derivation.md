# Task 1.1-H26: Persona Store & Entropy Gate — Gemini Derivation (Agent A)

## 1. Implementation Strategy

### 1.1 Persona Store (DDR-002)
- **Default Personas:** Create `personalities/` directory with `.md` files for `reviewer`, `planner`, `tiebreaker`, `operator`, and `classifier`.
- **Project Assignments:** Implement `.mbo/persona.json` schema to map roles to persona IDs.
- **User Library:** Update `callModel` to resolve persona prompts from `~/.orchestrator/personas/` (User), `./personalities/` (Project), or builtin `personalities/` (MBO).

### 1.2 Persona Injection (DDR-001)
- **`callModel` Update:** Prepend the resolved persona system prompt to the `systemPrompt` before `FIREWALL_DIRECTIVE`.
- **Persona Context:** Pass `personaId` via `options` to `callModel`.

### 1.3 Extraction Pass (DDR-001)
- **`Operator` Update:** Refactor `_safeParseJSON` to handle conversational responses when JSON is missing or malformed.
- **`_extractDecision`:** Implement a regex-based extraction pass in `Operator` to pull decisions (e.g., `APPROVED`, `REJECTED`, `PLAN`) from persona-styled text.

### 1.4 Entropy Gate (Section 27)
- **`Entropy Score` Logic:** Implement calculation in `Operator` (Critical=3, High=1.5, Low=0.5).
- **Hard Stop:** Enforce `status: blocked` and a hard stop if `Entropy Score > 10`.
