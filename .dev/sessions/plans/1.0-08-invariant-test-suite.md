# Plan: External Invariant Test Suite (Task 1.0-08)

## Objective
Implement a black-box test suite (`scripts/test-invariants.js`) that verifies all 11 security invariants defined in `SPEC.md` Section 22. This suite will be the final gate for Milestone 1.0.

## Key Files & Context
- `SPEC.md` Section 22: Source of truth for invariants.
- `scripts/verify-chain.js`: Tool for Invariant 4 (Event Store reproducibility).
- `bin/validator.py`: Tool for Invariant 10 & 11 (Entropy Tax).
- `src/`: Protected source code.

## Detailed Test Case Implementation
- **Invariant 1 & 2 (File Protection):** Verify `src/` permissions are 555 or 444 using `fs.statSync`. Attempt unauthorized write to `src/test.txt` and verify it fails.
- **Invariant 3 (Human Approval):** Verify `src/auth/operator.js` handles the `APPROVAL` event and that Stage 5 logic in `src/index.js` requires explicit `go`.
- **Invariant 4 (Event Store):** Execute `python3 scripts/verify-chain.py` and ensure the exit code is 0.
- **Invariant 5 (Rollback):** Verify `src/state/state-manager.js` contains snapshot/rollback logic via grep.
- **Invariant 6 (Firewall):** Grep `src/` for direct model provider imports (e.g., `openai`, `anthropic`) outside `call-model.js`.
- **Invariant 7 (Context Isolation):** Verify redaction logic in `src/auth/call-model.js` via grep.
- **Invariant 8 (Sandbox Secrets):** Verify environment sanitation in `src/sandbox/sandbox-manager.js` via grep.
- **Invariant 9 (Local Models):** Verify existence of `src/auth/local-detector.js`.
- **Invariant 10 (Entropy Tax):** Run `python3 bin/validator.py --all` and assert all files in `/src` pass.
- **Invariant 11 (Cell Bootstrapping):** Verify every `.js` file in `src/` contains the mandatory Entropy Tax header.

## Verification & Testing
- The suite must detect any current integrity failures.
- Each test case should include clear PASS/FAIL output.
