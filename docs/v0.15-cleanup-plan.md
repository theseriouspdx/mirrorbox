## Cleanup Sequence

Phases execute in order. No phase begins until the previous phase's stop condition passes.

---

### Phase 1 — Filesystem Cruft Removal

**Goal:** No untracked system artifacts in the working tree or repo history.

**Steps:**

1. **Purge `.DS_Store` files** — 35+ instances across root, bin/, src/, .dev/, audit/.
   - Check tracking: `git ls-files --cached | grep DS_Store`
   - Remove tracked: `git rm --cached $(git ls-files --cached | grep DS_Store)`
   - Remove from working tree: `find . -name .DS_Store -not -path './.git/*' -delete`

2. **Remove `scripts/scratch`** — empty file, not a test or operational script. Delete.

3. **Audit `.dev/bak/`** — backups accumulate per Invariant 13. Review entries older than
   the last two milestones. Archive pre-0.11.180 backups to `.dev/archive/` with a dated
   note. Do not delete without verifying nothing active references them.

4. **Resolve `src/cells/`** — contains only `logic.spec` (cell design constraint doc).
   Move `logic.spec` to `docs/cell-logic-spec.md` and remove the empty directory.

**Phase 1 stop condition:**
- `find . -name .DS_Store -not -path './.git/*' | wc -l` → `0`
- `git ls-files --cached | grep -i scratch` → empty
- `src/cells/` either documented or removed

### Phase 3 — Code Quality Pass

**Goal:** `src/` has no debug artifacts, dead code, or stylistic issues that would
distract an auditor.

**Steps:**

1. **Scan for debug artifacts:**
   `grep -rn "console\.log\|debugger" src/ --include="*.js" --include="*.ts" --include="*.tsx"`
   Keep structured/intentional log calls; remove ad-hoc debug noise.

2. **Scan for unlinked TODO/FIXME/HACK:**
   `grep -rn "TODO\|FIXME\|HACK" src/ --include="*.js" --include="*.ts" --include="*.tsx"`
   Any hit without a linked BUG or task ID must get a BUG entry or be resolved.

3. **Check import consistency in `src/auth/`** — core pipeline files (call-model, operator,
   did-orchestrator, model-router) must use consistent import style (CJS require() for .js,
   ESM import for TUI .tsx/.ts). No unintentional mixing.

4. **Move `src/cells/core/logic.spec`** per Phase 1 decision → `docs/cell-logic-spec.md`.

**Phase 3 stop condition:**
- Debug/TODO scans return zero unlinked hits.
- Import consistency confirmed across `src/auth/` and `src/graph/`.
- No ad-hoc debug output remains.

### Phase 4 — Scripts Audit & Organization

**Goal:** Every file in `scripts/` is either a passing test or a documented operational
tool. No dead or orphaned scripts.

**Steps:**

1. **Re-run the full test suite:** `node tests/run-all.js` — all scripts must exit 0.

2. **Categorize `scripts/`:**
   - **Test scripts** (`test-*.js`, `verify-*.js`): governed by TEST_STANDARDS.md
   - **Operational scripts** (`mbo-*.sh`, `alpha-*.js`, `generate-*.js`, etc.): must have
     a purpose header block

3. **Add headers to operational scripts** where missing:
   ```
   // PURPOSE: <one line>
   // USAGE: <invocation example>
   // OWNER: <claude|gemini|codex|operator>
   ```

4. **Verify `scripts/mbo-workflow-audit.js`** runs cleanly against current repo state.

5. **Evaluate `scripts/rebuild-mirror.js` and `scripts/rehash-events.js`** — if no longer
   needed, move to `.dev/archive/scripts/` with a dated reason note (per Invariant 15,
   non-destructive recovery — do not delete).

**Phase 4 stop condition:**
- `node tests/run-all.js` exits 0.
- All operational scripts have purpose headers.
- No script in `scripts/` is undocumented.
